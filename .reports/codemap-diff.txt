=== WHAT DROVE THE DIFF: the composition paint path (D71-D77) ===
17 commits (0e0f5e3..ab11391) added a second way to paint a board, behind the existing
Painter port. The prior maps had ZERO coverage of it.

  AutoPainter picks per board from config.painter.mode (auto | free | composition):
    - theme declares a `vocabulary`  -> COMPOSE: an LLM Composer fills a small structured
      order (section/group/photoBand); deterministic code in src/composition/ renders it
    - plain theme                    -> FREE-PAINT (the original path)
    - auto mode rescues to free paint if composing fails; `composition` mode fails loud

  Graph, packager, and QA are STRUCTURALLY UNCHANGED. Composedness is derived, never stored
  (isComposedHtml(state.html) via a data-composed marker). QA narrowly TRUSTS composed markup.
